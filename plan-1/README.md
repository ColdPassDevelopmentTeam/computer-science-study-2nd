# 1안: 공통 학습과 랜덤 발표

모든 참여자가 16주 동안 같은 질문 하나와 필수 키워드 3~5개를 공부하고, 주간 TIL과 랜덤
발표로 이해를 확인합니다. 목표는 결론부터 답하고 기본 개념의 동작과 전제를 자기 언어로
설명하는 것입니다. Depth 5~7 면접 훈련은 [2안](../plan-2/)에서 진행합니다.

참여자는 이 문서와 [16주 커리큘럼](./curriculum/)의 해당 주차 문서만 읽으면 됩니다.

## 한눈에 보기

| 항목 | 규칙 |
|---|---|
| 기간 | 완료한 16개 주차 세션, 4주마다 범위 점검 |
| 공통 범위 | 질문 1개, 한 문장 결론, 키워드 3~5개, 설명 순서와 제외 범위 |
| 필수 준비 | 개인 블로그 또는 GitHub의 주간 누적 TIL 1개 |
| 공유 마감 | 세션 시작 1시간 전까지 해당 주 Discord 공지 스레드에 공유 |
| 발표 | 참석자 중 1~2명을 랜덤 순환으로 선정, 한 사람당 5~10분 |
| 자유 질문 | 희망자만 참여, 최대 20분, 질문이 없으면 생략 |
| 세션 | 30~60분 권장, 최대 90분 |
| AI | 전 과정 사용 가능, 핵심 사실 검증과 직접 설명은 본인 책임 |
| 세션 후 | 필수 산출물 없음, 기존 TIL 수정은 선택 |

## 이번 주 참여 방법

1. 공지된 질문, 필수 키워드 3~5개와 제외 범위를 확인합니다.
2. 월요일부터 세션 전날까지 각자 공부하며 TIL 한 곳에 짧게 누적합니다.
3. 세션 시작 1시간 전까지 TIL 링크 또는 최소 항목 세 가지를 Discord에 공유합니다.
4. 세션 시작 후 발표자 1~2명을 추첨하고, 선정된 사람은 자신의 TIL로 설명합니다.
5. 발표 뒤 궁금한 사람이 있을 때만 자유롭게 질문하거나 의견을 나눕니다.

별도 슬라이드, 발표 대본과 스터디 레포용 학습 문서는 만들지 않습니다. 세션이 취소되면
보강 과제 없이 다음 일정에 같은 주차를 진행합니다.

## 주간 TIL

TIL은 매일 새 글을 만드는 대신 **한 주에 페이지 또는 파일 하나**를 만들고 공부한 날마다
몇 줄씩 누적합니다.

| 구분 | 기준 |
|---|---|
| 위치 | 개인 블로그 또는 개인 GitHub |
| 최소 항목 | 내 한 문장 답변, 설명 순서, 헷갈리는 점 |
| 선택 항목 | 출처, 동작 원리, 사례, 코드, 다이어그램 |
| 별도 편집 | 주당 20분을 넘기지 않음 |
| 평가하지 않는 것 | 글 길이, 문장력, 디자인, 방문자 수, 커밋 수 |

```markdown
# 이번 주 질문

- 내 한 문장 답변:
- 답변을 설명하는 순서:
- 헷갈리는 점:
- 확인한 출처(선택):
```

링크를 다른 참여자가 열 수 없다면 최소 항목 세 가지만 Discord 공지 스레드에 붙여 넣습니다.
세션을 위해 글을 다시 쓰거나 별도 요약본을 만들 필요는 없습니다.

TIL은 필수 준비이지만 미작성자를 추첨에서 제외하거나 밀린 글을 소급 제출시키지 않습니다.

## 주말 세션

| 구간 | 활동 |
|---|---|
| 5분 | 이번 주 범위 확인과 발표자 랜덤 추첨 |
| 10~20분 | 발표자 1~2명의 TIL 기반 즉석 발표 |
| 선택, 최대 20분 | 원하는 참여자의 자유 질문과 의견 공유 |
| 5분, 점검 주차 10분 | 확인된 내용과 다음 주 질문 확인 |

- 발표자는 TIL을 읽기보다 결론과 동작 흐름을 자기 언어로 설명합니다.
- AI가 만든 내용을 포함해 자신이 적은 핵심 주장을 설명할 책임이 있습니다.
- 모르는 내용, 가정과 직접 확인한 사실을 구분합니다.
- 질문에 바로 답하지 못하면 추측하지 않고 확인 방법을 제시합니다.
- 질문이 없거나 답하지 못한 질문이 있어도 발표가 실패한 것으로 보지 않습니다.

발표와 자유 질문이 끝나면 정해진 시간을 채우지 않고 종료합니다. 질문 개수를 채우거나
돌아가며 발언하지 않으며, 질문이 남아도 90분을 넘기지 않습니다.

### 자율 질문

질문 참여와 개수는 전적으로 자율입니다. 스터디장은 참여자를 지목하거나 질문 순서를
돌리지 않습니다. 질문할 때는 다음 중 실제로 궁금한 한 가지만 확인합니다.

1. 자신의 조사와 다른 주장
2. 빠진 동작 과정이나 전제
3. 두 설명의 조건 차이

한 발표자에게 Depth 5~7 꼬리질문을 이어가지 않습니다.

## 발표자 랜덤 순환

1. [루트 참여자 표](../README.md#참여자)의 `참여 확정` 인원을 섞어 후보 목록을 만듭니다.
2. 세션 시작 후 참석자 중 1~2명을 선정하고, 발표한 사람은 한 바퀴가 끝날 때까지 제외합니다.
3. 결석자는 후보에 남기며, 모든 활성 참여자가 발표하면 다시 전체를 섞습니다.
4. 2주 연속 결석하면 비공개로 참여 방식을 확인하고, 3주 연속이면 `휴면`으로 전환합니다.
5. 복귀 의사를 알리면 당일 추첨 전에 현재 후보 목록의 마지막에 다시 넣습니다.

### 패스와 준비 예외

- 1~4, 5~8, 9~12, 13~16주차마다 이유를 밝히지 않고 1회 패스할 수 있습니다.
- 패스한 사람은 당일 추첨에서 제외하고 다음 세션 후보에 남으며, 별도 기여를 요구하지 않습니다.
- 건강이나 긴급 업무 사정에는 패스 횟수를 적용하지 않습니다.
- 준비가 부족하면 5~10분 발표 대신 `현재 이해 1분 + 막힌 지점 1개`만 설명할 수 있습니다.
- TIL을 작성하지 않아도 추첨에서 제외하지 않고 현재 이해와 막힌 지점을 설명합니다.
- 미작성 TIL은 소급 제출하지 않으며, 2주 연속 미작성일 때만 비공개로 운영 방식을 조정합니다.
- 취소된 세션은 완료 주차나 패스 기간에 포함하지 않습니다.

## AI 사용

AI는 자료 탐색, 초안·요약, 코드·예제, 예상 질문과 발표 연습에 사용할 수 있습니다. 모델명과
프롬프트는 제출하지 않습니다. 핵심 사실은 공식 문서나 신뢰할 수 있는 자료로 확인하고,
TIL에 적은 내용은 직접 설명할 수 있어야 합니다.

## 자유 모각코

평일 20시 모각코는 가능한 사람이 여는 선택적 Discord 공간입니다.

- 자유롭게 들어오고 나갈 수 있으며 참석 횟수와 체류 시간을 기록하지 않습니다.
- 사전 신청, 불참 사유와 대체 인증을 요구하지 않습니다.
- 카메라, 음성 사용과 화면 공유는 선택입니다.
- 모각코 불참은 주간 학습이나 발표자 선정에 영향을 주지 않습니다.
- 회사 코드, 개인정보와 비공개 자료를 화면이나 TIL에 노출하지 않습니다.

## 스터디장 책임

- 다음 질문, 필수 키워드 3~5개와 세션 시각 공지
- TIL을 공유할 Discord 공지 스레드와 발표 순환 목록 준비
- 세션 시작 후 발표자 추첨과 시간 관리
- 질문이 나온 경우 사람 대신 주장과 근거를 검토하도록 대화 정리
- 공지한 필수 범위를 늘리거나 여러 주차를 합치지 않음

## 세션 후

- 별도 회고 문서와 추가 발표 자료 없음
- 확인된 오류를 기존 TIL에 반영하는 것은 선택이며 별도 마감 없음
- 끝내지 못한 범위를 다음 주 의무 과제로 넘기지 않음
- TIL 미작성이 2주 연속이면 공개 지적 없이 참여 방식을 함께 조정
