# 2안: 고강도 기술 면접과 오답 보충

공개된 범위를 공부한 뒤 실제 질문은 미리 알지 못한 상태에서 답하고, 답변의 한 주장을
Depth 5~7까지 검증하는 4주 과정입니다. 사람의 능력이나 등급이 아니라 답변 속 **주장**만
분류하고, 검증된 핵심 오답만 짧게 보충합니다.

참여자는 이 문서, [4주 커리큘럼](./curriculum/)의 Depth 기준과 해당 주차 문서만 읽으면 됩니다.

## 시작 조건

- 공개 질문과 키워드를 자료 없이 2~3분 설명할 수 있어야 합니다.
- 모든 참여자가 실시간 답변 중 자료·검색·AI를 사용하지 않는 규칙에 동의해야 합니다.
- 처음 접하는 개념이 대부분이라면 [1안](../plan-1/)을 먼저 진행합니다.
- 1안과 2안을 같은 세션에서 섞지 않습니다.

4주는 완료한 4개 세션을 뜻합니다. 세션이 취소되면 보강 과제 없이 다음 일정에 같은 주차를
진행합니다.

## 한눈에 보기

| 항목 | 규칙 |
|---|---|
| 공통 범위 | 대표 질문 1개, 키워드 3~5개, 시스템 가정과 제외 범위 |
| 실제 질문 | 질문자 1~2명이 전체 최대 2개 준비, 정확한 문장은 비공개 |
| 질문 마감 | 24시간 전 제출, 12시간 전 스터디장 검토 완료 |
| 면접 대상자 | 질문자를 제외한 참석자 중 1~2명 랜덤 순환 선정 |
| 답변 | 첫 답변 2~3분, 공개 시스템 적용 후 한 주장을 Depth 5~7까지 확인 |
| AI | 사전 준비와 검증에는 허용, 실시간 답변 중 금지 |
| 검증 | 공식 문서, 코드, 로그 또는 작은 실험 사용 |
| 기록 | 평소 TIL 없음, 배정된 핵심 오답만 보충 |
| 오답 보충 | 세션 전체 최대 2개, 종료 후 48시간 안에 공유 |
| 세션 | 90분 확보, 60분부터 종료 가능, 최대 120분 |

## 매주 진행

| 시점 | 할 일 |
|---|---|
| 직전 세션 종료 전 | 대표 질문·키워드·시스템 가정·제외 범위·질문자·세션 시각 공지 |
| 월요일~세션 전날 | 모든 참여자 공개 범위 학습, 질문자는 실제 질문 준비 |
| 세션 24시간 전 | 질문자가 질문·필수 답변 요소·Depth 5~7 방향·검증 방법을 비공개 제출 |
| 세션 12시간 전 | 스터디장이 질문의 범위와 검증 가능 여부 확인 |
| 세션 시작 | 면접 대상자 추첨 후 실제 질문 공개 |
| 세션 중 | 첫 답변, 공개 시스템 적용, Depth 5~7 질문, 근거 검증, 주장 분류 |
| 세션 종료 후 48시간 | 배정된 핵심 오답이 있는 참여자만 보충 링크 공유 |

## 역할

### 스터디장

- 대표 질문, 필수 키워드 3~5개, 시스템 가정, 제외 범위와 질문자 공지
- 질문이 공개 범위와 검증 조건을 만족하는지 12시간 전까지 확인
- 면접 대상자 순환 목록과 세션 시간 관리
- 사람 대신 주장만 분류하고 핵심 오답을 세션 전체 최대 2개 선정

### 질문자

- 공개 범위 안에서 조건을 바꾼 질문을 전체 합계 최대 2개 준비
- 첫 답변의 필수 요소와 Depth 5~7 꼬리질문 준비
- 질문의 전제, 예상 답변, Depth 5~7 방향과 확인 방법을 직접 설명
- 실제 질문은 세션 전까지 대상자에게 공개하지 않음

질문자는 해당 주 면접 대상자에서 제외하지만 다음 주 후보 목록에는 남습니다. 역할은 순환하며,
대체할 사람이 있다면 같은 사람이 연속으로 맡지 않습니다. 예상 참석자 중 면접 대상자가 최소
한 명 남도록 질문자 수를 정합니다.

### 모든 참여자

- 대표 질문과 필수 키워드 3~5개의 원리와 판단 기준 학습
- 사전 학습에는 AI를 사용할 수 있지만 핵심 사실과 출처를 직접 확인
- 면접 대상자가 아니어도 반례, 다른 전제와 검증에 참여
- 회사 코드, 개인정보와 비공개 장애 정보를 공개 사례로 사용하지 않음

### 면접 대상자

- 첫 답변을 자료와 AI 없이 2~3분 안에 설명
- 결론, 동작 과정, 조건과 모르는 부분을 구분
- 꼬리질문에서 대안, 비용, 실패, 검증과 운영 책임까지 설명
- 모르는 사실은 추측하지 않고 `모르겠습니다` 또는 `확인 필요`로 구분
- 검증 단계에서는 다른 참여자와 함께 공식 문서와 코드 확인
- 배정된 핵심 오답만 48시간 안에 보충

## 질문 준비

스터디장은 직전 세션이 끝나기 전에 다음 형식으로 공개 범위를 공지합니다.

```markdown
## 이번 주 기술 면접

- 대표 질문:
- 필수 키워드 3~5개:
- 가정할 시스템 또는 상황:
- 이번 주 제외 범위:
- 질문자 1~2명:
- 주말 세션 시각:
```

질문자는 실제 질문, 첫 답변의 필수 요소, Depth 5~7 방향과 검증 방법을 개인 메모에 준비해
24시간 전까지 제출합니다. 다음 조건을 모두 만족하고 12시간 전까지 확인된 질문만 사용합니다.

- 필수 키워드 3~5개 안에서 시작
- 짧은 암기 답변으로 끝나지 않음
- Depth 5 이상의 대안과 비용까지 연결
- 공식 문서나 실행 결과로 핵심 주장 확인 가능
- 특정 회사 경험이나 비공개 지식이 없어도 답변 가능

평일 모각코는 자유 참여입니다. 횟수와 접속 시간을 기록하거나 대상자 선정과 오답 분류에
사용하지 않으며, 불참을 주말 세션 불참 사유로 보지 않습니다.

## 면접 대상자 랜덤 순환

1. [루트 참여자 표](../README.md#참여자)의 `참여 확정` 인원을 섞어 후보 목록을 만듭니다.
2. 세션 시작 후 참석자 중 이번 주 질문자를 제외하고 1~2명을 선정합니다.
3. 선정된 사람은 한 바퀴가 끝날 때까지 후보에서 제외하고, 결석자는 후보에 남깁니다.
4. 모든 활성 참여자가 선정되면 전체 후보를 다시 섞습니다.
5. 3주 연속 주말 결석하면 `휴면`으로 전환하고, 복귀 시 현재 후보의 마지막에 넣습니다.

- 4주 과정마다 이유를 밝히지 않고 1회 패스할 수 있으며 이름은 후보에 남습니다.
- 패스한 사람은 질문, 반례 또는 검증으로 참여합니다.
- 건강이나 긴급 업무 사정에는 패스 횟수를 적용하지 않습니다.
- 취소된 세션은 완료 주차나 패스 기간에 포함하지 않습니다.
- 질문자를 제외한 대상자가 한 명도 남지 않으면 질문을 공개하지 않고 같은 주차를 다시 진행합니다.

## 주말 세션

### 시간과 순서

- 확보 시간: 90분
- 종료 가능: 시작 후 60분부터
- 절대 상한: 120분
- 면접 대상자: 1~2명
- 핵심 질문: 세션 전체 최대 2개

대상자가 한 명이면 질문을 최대 2개, 두 명이면 원칙적으로 한 명당 질문 1개를 사용합니다.
90분을 넘길 계획이면 시작 전에 전원이 동의해야 하며, 동의해도 120분을 넘기지 않습니다.

| 시간 | 활동 |
|---:|---|
| 5분 | 범위 확인, 면접 대상자 선정, 안전 원칙 확인 |
| 5~10분 | 자료와 AI 없는 첫 답변 |
| 5~10분 | 공개된 시스템 조건에 답변 적용 |
| 20~35분 | 한 주장에 대한 Depth 5~7 꼬리질문과 반례 |
| 15~20분 | 공식 문서, 코드, 로그 또는 실험으로 핵심 주장 검증 |
| 5~10분 | 주장 분류, 핵심 오답 선정과 개선 답변 재시도 |

60분 세션에서는 질문과 검증을 하나씩 우선합니다. 두 번째 질문을 열어 첫 번째 질문의 검증
시간을 줄이지 않습니다.

### 자료와 AI 사용

| 단계 | 면접 대상자 | 질문자와 다른 참여자 |
|---|---|---|
| 사전 학습·질문 준비 | AI 사용 가능, 핵심 내용을 직접 설명 | AI 사용 가능, 질문의 전제와 범위 확인 |
| 첫 답변 | 자료·검색·AI 사용 금지 | 질문지만 확인 가능 |
| 꼬리질문 | 자료·검색·AI 사용 금지 | 준비한 질문과 메모 사용 가능 |
| 검증 | 공식 문서·코드·로그·검색·AI 사용 가능 | 검색·AI를 포함해 함께 확인하고 실행 가능 |
| 세션 후 | AI 사용 가능 | AI 사용 가능 |

휴대전화와 노트북을 치우도록 강제하거나 사용 내역을 감시하지 않습니다. 단계가 바뀔 때
허용 범위를 알리고 자율적으로 지킵니다.

## 주장 분류

| 분류 | 기준 | 처리 |
|---|---|---|
| 정확 | 범위와 조건을 포함해 근거와 일치 | 유지 |
| 부분 정확 | 방향은 맞지만 중요한 조건·과정이 빠지거나 과도하게 일반화 | 조건 보완 |
| 오류 | 공식 문서나 재현 가능한 결과와 충돌 | 핵심이면 오답 보충 후보 |
| 보류 | 세션 안에 판별할 근거가 부족 | 미확인 상태로 종료 가능 |

분류 결과를 사람의 속성이나 종합 등급으로 표현하지 않습니다.

## 48시간 오답 보충

검증된 핵심 오답만 세션 전체에서 최대 2개 선정합니다. 오류가 없으면 보충하지 않습니다.
두 명이 답했다면 하나씩 맡길 수 있고 한 사람에게 두 개가 배정될 수도 있습니다. 부분 정확한
주장은 빠진 조건 때문에 실제 판단이 달라지는 경우에만 포함합니다.

```markdown
## 오답 1

- 오답:
- 정정 근거:
- 개선 답변:
```

- 종료 후 48시간 안에 개인 블로그 또는 GitHub에 추가하고 Discord 공지 스레드에 링크 공유
- 글의 분량과 디자인은 평가하지 않음
- 공식 문서, 코드, 로그 또는 재현 결과를 직접 확인
- AI를 사용할 수 있지만 개선 답변은 자료 없이 다시 설명
- 비공개 정보는 일반화한 재현 사례로 변경
- 기한을 지키기 어려우면 스터디장에게 비공개로 알림
- 공개 독촉, 벌점과 밀린 오답 일괄 제출 금지

## 운영 안전장치

- 개인 점수, 순위, 합격·불합격과 레벨 판정 금지
- 오답 수, 질문 수와 답변 속도로 사람을 비교하지 않음
- 랜덤 선정을 벌칙이나 망신 수단으로 사용하지 않음
- 모르는 답을 억지로 추측하게 만들지 않음
- 질문자의 예상 답변을 유일한 정답으로 취급하지 않음
- 공식 문서보다 연차, 자신감과 말의 유창함을 우선하지 않음
- 세션에서 끝내지 못한 모든 내용을 추가 과제로 배정하지 않음
- 동의 없이 녹음하거나 답변을 외부에 공개하지 않음
